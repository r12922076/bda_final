# Evidence Methodology

The evidence dataset is directional rather than a representative survey. It is intended to support a defensible demand hypothesis for a course prototype.

## Collection categories

1. Official TradingView documentation.
2. Public user-pain discussions.
3. Adjacent-product pricing pages.
4. Technical architecture references.
5. Platform, privacy, and investment-advice risk references.

## Coding rules

Each evidence row is assigned a source type, pain category, relevance note, and strength rating. Official documentation is treated as strong technical evidence, but not as direct willingness-to-pay evidence. Competitor pricing is treated as willingness-to-pay benchmark evidence, not as a direct demand curve. Public discussions are qualitative evidence and should not be described as representative.

## Exclusions

The dataset excludes private pages, login-only content, paywalled material, unauthorized scraping of private scripts, and long copyrighted excerpts.

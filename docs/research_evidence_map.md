# Research Evidence Map

The report uses evidence in four roles.

1. **Official technical evidence**: TradingView documentation on repainting, strategies, alerts, and webhooks motivates the validation problem.
2. **Public user-pain evidence**: public discussions show qualitative pain around repainting, webhook issues, backtest/live mismatch, and paid-script trust.
3. **Competitor benchmark evidence**: adjacent paid products support a plausible willingness-to-pay corridor.
4. **Architecture and governance references**: static hosting, webhook best practices, event-driven architecture, data minimization, and investment-advice boundaries support the technical and risk design.

The evidence is intentionally interpreted conservatively. It supports a plausible demand hypothesis, not a measured demand curve.

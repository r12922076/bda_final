# Risk Register

| Risk | Control |
|---|---|
| Platform dependency | Use user-authorized alert payloads and user-provided exports. Avoid unauthorized scraping. |
| Investment-advice boundary | Provide workflow diagnostics only. Do not provide personalized trade recommendations or execute trades. |
| Privacy and data minimization | Do not collect broker passwords, trading API keys, or unnecessary personal data. |
| Data cold start | Begin with one-time health checks, then convert users into monitoring after alert history accumulates. |
| Competitive imitation | Differentiate through audit trails, evidence-backed reports, and accumulated alert/outcome history. |
